# webinar-recorder

Локальный пайплайн **запись → транскрипция → саммари** для онлайн-встреч и вебинаров на **macOS**. Всё считается локально, без облака и без подписок на транскрибацию: данные не покидают твой Mac.

- 🎙 Захват системного звука (Zoom / любой плеер) через виртуальное аудио **BlackHole**
- 🗣 Распознавание речи локально на GPU через **whisper.cpp** (модель large-v3-turbo, русский и др.)
- 📝 Структурированный конспект в **Markdown**; PDF можно собрать отдельной командой

---

## Требования
- macOS (Apple Silicon или Intel)
- [Homebrew](https://brew.sh)
- ~2 ГБ места под модель
- Для шага саммари — установленный и залогиненный CLI `claude` **или** `codex` (опционально; без них получишь только транскрипт)

## Установка
```bash
git clone <этот-репозиторий> webinar-recorder
cd webinar-recorder
./setup.sh          # ffmpeg + whisper-cpp + BlackHole + скачивание модели
```

## Настройка звука (один раз, вручную)
macOS не даёт писать системный звук напрямую — нужен «двойной выход», чтобы ты **и слышал**, и **записывал**:

1. Открой **Audio MIDI Setup** (Spotlight → «Audio MIDI Setup»).
2. Внизу слева **«+» → Create Multi-Output Device**.
3. Поставь галочки на **двух** устройствах: то, через что слушаешь (наушники/динамики) **и** **BlackHole 2ch**.
4. Устройство, через которое слушаешь, оставь **Master/Primary**; у **BlackHole** включи **Drift Correction**.

   | Устройство | Use | Master | Drift Correction |
   |---|---|---|---|
   | Твои наушники/динамики | ✅ | ✅ | ⬜ |
   | BlackHole 2ch | ✅ | ⬜ | ✅ |

5. **System Settings → Sound → Output → выбери Multi-Output Device.**
   (Громкость с клавиатуры при этом не работает — выставь её заранее в Zoom.)

## Использование
### Локальный UI
```bash
python3 ui.py
```

Открой `http://127.0.0.1:8787`. В интерфейсе доступны три режима:
- загрузить готовый аудио/видео файл;
- записать офлайн-встречу с микрофона;
- записать Zoom / системный звук через BlackHole.

Сценарии в UI:
- **Готовая запись**: выбери файл → нажми **Транскрибировать файл** → дождись ссылку `transcript.txt`.
- **Офлайн встреча**: нажми **Начать** → после встречи **Остановить** → **Обработать запись**.
- **Zoom / системный звук**: сначала настрой BlackHole и Multi-Output Device, проверь `./test-capture.sh`, затем в UI нажми **Начать** → **Остановить** → **Обработать запись**.

UI работает поверх тех же скриптов `record.sh`, `record-mic.sh`, `process.sh`. Консольный запуск остаётся доступен.

### Консоль
```bash
# 1) Проверь захват ДО встречи (включи любое видео со звуком):
./test-capture.sh                 # ждём «✅ Звук захватывается»

# 2) Во время встречи (caffeinate не даёт Mac уснуть):
caffeinate -i ./record.sh         # Ctrl+C в конце

# 3) После встречи — транскрипция + саммари:
./process.sh                      # берёт последнюю сессию
# или конкретную:  ./process.sh webinar_20260625_1700
```

Результаты — в `out/<сессия>/`:
- `transcript.txt` — полная расшифровка
- `summary.md` — конспект, если установлен и залогинен `claude` или `codex` CLI

### Язык
По умолчанию русский. Другой язык:
```bash
WHISPER_LANG=en ./process.sh
```

## Готовый аудиофайл
Если запись уже есть файлом (`m4a`, `mp3`, `wav`, `mp4` и т.п.), его можно сразу отдать в тот же пайплайн. Файлы диктофона iPhone/macOS обычно экспортируются как `.m4a`; этот формат поддерживается через `ffmpeg`.

```bash
./process.sh ~/Downloads/lecture.m4a
# или с понятным именем сессии:
SESSION=my_lecture ./process.sh ~/Downloads/lecture.m4a
```

Скрипт создаст новую папку `out/audio_<имя>_<дата>/`, приведёт файл через `ffmpeg` к формату **16 кГц mono WAV** кусками по 30 минут (`chunk_000.wav`, `chunk_001.wav`, ...), затем запустит обычную транскрипцию и саммари.

### PDF
`process.sh` делает только `.md`. Для PDF нужен `pandoc` (`brew install pandoc`) — конвертация идёт через headless Google Chrome:
```bash
pandoc summary.md -s -o summary.html
"/Applications/Google Chrome.app/Contents/MacOS/Google Chrome" \
  --headless --no-pdf-header-footer --print-to-pdf=summary.pdf "file://$PWD/summary.html"
```

## Офлайн-лекция (запись с микрофона)
Если звук идёт не из системы (Zoom), а **вживую в аудитории** — нужен микрофон, а **BlackHole и Multi-Output Device НЕ нужны**. Установка: достаточно `ffmpeg` + `whisper-cpp` + модель (BlackHole можно не ставить).

```bash
./record-mic.sh --list        # показать аудио-входы (найти индекс нужного микрофона)
./record-mic.sh --test        # 8 сек проверки уровня перед лекцией
caffeinate -i ./record-mic.sh # запись (авто-выбор встроенного микрофона), Ctrl+C в конце
MIC=2 ./record-mic.sh         # принудительно внешний микрофон с индексом 2
./process.sh                  # дальше всё так же: транскрипт + саммари
```

**Советы для аудитории:**
- 🎤 На 3–4 часа в комнате **сильно лучше внешний микрофон** (петличка на преподавателе или USB-конференц-микрофон вроде Jabra/Blue Yeti), чем встроенный — он берёт речь чисто, без гула помещения. Встроенного хватает только для тихой небольшой комнаты с ноутбуком рядом с говорящим.
- 🔌 Возьми зарядку — 4 часа записи сажают батарею; `caffeinate -i` не даёт Mac уснуть.
- 🔐 Первый запуск попросит доступ к микрофону → **Allow** (или System Settings → Privacy & Security → Microphone → включи Terminal).
- 🗣 Whisper **не различает спикеров** (нет диаризации): лекцию одного преподавателя расшифрует отлично, тихие вопросы студентов издалека могут теряться — отсюда снова совет про внешний микрофон.

## Как это работает
- `record.sh` — находит BlackHole в `ffmpeg avfoundation`, пишет **16 кГц моно** кусками по 30 мин (`chunk_NNN.wav`) — при сбое теряется максимум последние полчаса.
- `record-mic.sh` — то же, но с микрофона (для офлайн-лекций, без BlackHole).
- `process.sh` — если передан готовый аудиофайл, сначала импортирует его в `chunk_NNN.wav`; затем гоняет каждый кусок через `whisper-cli` → склеивает `transcript.txt` → отдаёт в `claude`/`codex` CLI для конспекта.
- `process.sh` сначала использует Metal/GPU. Если `whisper-cli` падает на Metal/GPU, он автоматически повторяет распознавание на CPU (`-ng`) с низким приоритетом (`nice`) и 2 потоками по умолчанию. Это не меняет качество модели, только снижает нагрузку аварийного fallback-режима.

## Известные ограничения
- ⚠️ **Потеря ~10–15% сэмплов** при длинной записи (дрейф тактовой частоты BlackHole vs ресемплер в ffmpeg). На транскрипт влияет мелкими пропусками слов; для конспекта обычно некритично. *(TODO: писать в нативные 48 кГц без ресемплинга на лету, понижать до 16 кГц уже при транскрипции.)*
- Нужен ручной шаг с Multi-Output Device (см. выше) — автоматизировать средствами CLI нельзя.

## Траблшутинг
- **`record.sh`/`test-capture.sh` молча выходят или «Input/output error»** → нет доступа к микрофону: **System Settings → Privacy & Security → Microphone → включи свой Terminal** и запусти заново (BlackHole для macOS = «микрофон»).
- **«BlackHole не найден»** → не сделал `sudo killall coreaudiod`, либо нужен ребут.
- **Тишина в `test-capture.sh`** → в System Settings → Sound → Output не выбран Multi-Output Device, или звук реально не играет.
